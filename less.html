Ogólne

Zbiór podstawowych styli znajduje się w pliku /LayoutBundle/.../style.less.
Ładowane są tam niezbędne moduły bootstrapa, rozszerzające je pliki stworzone przez nas oraz inne style, które są przeznaczone do użytku globalnego.

  • style piszemy generycznie, czyli tak aby były możliwe do użycia w szerokim polu zastosowań i w różnym kontekście
  • jako obowiązującą jednostkę miary przyjmujemy px - jednostek względnych (% i em) używamy tylko w uzasadnionych przypadkach
  • nie piszemy styli w atrybucie style danego tagu (z wyłączeniem mechanizmów JavaScript)
  • unikamy zapisywania reguł CSS w tagu <style>, bezpośrednio w szablonie
  • w miarę możliwości tworzymy sprite’y z elementów layoutu, aby zmniejszyć liczbę request’ów
  • o ile to możliwe, korzystamy z wcześniej zdefiniowanych zestawów reguł i nie powielamy kodu odpowiedzialnego za te same fragmenty layoutu

Dziedziczenie

Jeśli chcemy zmienić domyślny wygląd elementów bootstrapowych, robimy to wyłącznie w pliku rozszerzającym, o takiej samej nazwie jak modyfikowany moduł, bezpośrednio w katalogu /LayoutBundle/Resources/public/less/. Nie dotykamy oryginalnych plików bootstrapa. Never.

Aby korzystać z funkcji skracających niektóre długie zapisy w CSS (głównie na potrzeby obsługi kilku przeglądarek) np. .border-radius(5px); trzeba załączyć podstawowy zestaw LESSów do arkusza, czyli: 

@import "bundles/layout/less/style.less";

W ten sam sposób możemy załączać arkusze właściwe dla różnych modułów na stronie.

Wspomniany wcześniej arkusz zawiera m.in. właśnie zestaw funkcji, zwanych w LESS mixins’ami - plik bootstrap/mixins.less

Formatowanie

#login-modal {
    @headingColor: #000;   <-  zmienne

    .border-radius(5px);   <-  mixiny

    background: #fff;      <-  "normalne" reguły
    width: 300px;

    h2 {			     <- zagnieżdżenie
        color: @headingColor;

        span {
            padding-left: 10px;
        }
    }

    &:before {		     <- odwołanie w kontekście bieżącego elementu
        margin-top: 5px;
    }
}

Kod formatujemy w sposób przedstawiony powyżej, z zachowaniem nowych linii i kolejności. Dodatkowo należy pamiętać o wydajności pisanych selektorów - parsery CSS czytają selektor od prawej do lewej!

selector#login-modal > div > form > *:first-child > input[type="text"] <- wolny

#login-modal fieldset:first-child input[type=”text”] <- szybki
